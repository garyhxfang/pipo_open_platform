# 鉴权与签名

## 请求头规范

每次请求必须包含以下 Header：

- `X-App-Id`：应用 ID
- `X-Merchant-Id`：商户号
- `X-Timestamp`：ISO8601 时间，误差不超过 5 分钟
- `X-Signature`：签名值

## 签名步骤

1. 按字段名升序拼接参与签名的参数
2. 使用 `&` 连接为待签名字符串
3. 使用商户私钥做 `RSA-SHA256` 签名
4. 对签名结果进行 Base64 编码后放入 `X-Signature`

## 验签失败常见原因

1. 参与签名字段不完整
2. 时间戳过期
3. 使用了错误密钥
4. 请求体经过网关改写导致签名串不一致
